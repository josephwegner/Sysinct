Your logs are stuck on the server. Set them free!

In fact, it's really easy. Point your syslog drain at a sysinct server, and then read it all with socket.io!

Seriously. I'll show you. Right here>.

I assume you've got some Heroku server running something. If you don't, make one. Now on the command line, run this:

heroku drains:add http://sysinct.herokuapp.com/drains/REPLACETHIS

Now watch your logs show up:

You should probably remove that drain, once you're sufficiently wowed:

heroku drains:remove http://sysinct.herokuapp.com/drains/REPLACETHIS

Magic, right? The cool thing is that the code to do that is incredibly simple. It looks like this:


<script src="http://sysinct.herokuapp.com/socket.io/socket.io.js"></script>
<script type="text/javascript">
  var socket = io.connect("http://sysinct.herokuapp.com");
  socket.emit('subscribe', 'yourapp');
  socket.on('log', function(data) {
    console.log('log', data);
  });
</script>

Now anything that drains to http://sysinct.herokuapp.com/drains/yourapp will show up in those logs.

So go ahead. Start logging. Use whatever key you want - just tell your syslog to drain to http://sysinct.herokuapp.com/drains/WHATEVERYOUWANT

Privacy & Security

Sysinct is open source. You can see the code if you want, and you can pretty clearly see that I'm not at all interested in storing your logs. I send them to the websocket, and then I'm done with it.

Don't trust me? Fork the repo, and host your own sysinct server. It's stupid simple.
